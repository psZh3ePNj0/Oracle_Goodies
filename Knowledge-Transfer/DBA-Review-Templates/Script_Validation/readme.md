<h1>SCRIPT EXECUTION VALIDATION REVIEW:</h1> 


$$\color{blue} Both \space syntax \space and \space semantics \space based,$$ SQL scripts should be validated on:

- If the scripts completed with no errors $$\color{blue}and$$
- $$\color{blue}The \space logic, \space intent \space and \space business \space process \space impact \space within.$$
  For example, volume and scope considerations; no one wants a script unexpectedly deleting hundreds of thousands of rows......
